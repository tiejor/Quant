# 因子定义为 Python 函数，而非配置文件或 DSL

因子用纯 Python 函数表达，一个因子一个函数。因子定义文件 `factors.py` 就是代码，不需要配置解析层。

**理由：** 数学系背景的用户写 `return close.pct_change(20)` 和写数学公式一样自然。配置文件（YAML/JSON）要求预定义有限算子集合，新因子类型需要改代码来扩展算子——这反而比直接写函数更麻烦。DSL 需要额外的解析器维护成本，对个人研究项目是过度设计。

**放弃的方案：**
- YAML 配置：限制因子只能组合预设算子，扩展需要同时改配置和算子代码
- 公式字符串 / DSL：需要实现健壮的表达式解析器，维护成本高于收益

**后果：** 因子函数签名不强制约束，错误原本在运行时暴露。通过单元测试全覆盖（边界值、异常输入、签名校验）在加载前捕获，几十个函数的规模下测试开销可接受。
