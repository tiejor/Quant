# 数据存储选型：DuckDB

行情和基本面数据存储在本地 DuckDB 单文件中，不依赖外部数据库服务。

**理由：** DuckDB 是嵌入式列存分析引擎，零配置、零运维。千万级日线数据在笔记本上查询秒级返回。与 pandas 通过 `duckdb.sql().df()` 双向互转，无序列化开销。个人因子研究不需要多用户并发、权限管理、备份恢复这些数据库运维功能。

**放弃的方案：**
- SQLite：同样零配置，但是行存，分析查询（聚合、窗口函数）比列存慢 10-100 倍
- PostgreSQL：需要安装、配置、管理服务进程，对单用户本地研究是运维噪音
- 纯 Parquet/CSV 文件：缺乏 SQL 查询能力，复杂过滤和 join 需要手动写 pandas 代码

**后果：** DuckDB 并发写入能力弱（单写），但对单人本地研究不构成限制。若未来需要多人共享数据，可迁移到 PostgreSQL，pipeline.py 只需替换连接层。
